# HdLabs.Down (EzLabs Down)

**URL을 넣고 분석(예정)**하는 자리에 맞춘 WPF 셸입니다. 현재는 URL 입력 + “다음 단계” 안내 수준의 **프로토타입**에 가깝습니다.

## 프로젝트 분석

- **동작:** `Analyze` 시 URL이 비어 있지 않으면 “Would analyze: …” 메시지로 **플레이스홀더**
- **다음:** HTTP 헤더 조회, MIME·크기, 실제 다운로드/큐, yt-dlp 연동 등이 후보

## 기능 점수 (10점 만점)

| 항목 | 점수 |
|------|------|
| **현재 완성도** | **3** |
| **잠재(아키텍처)** | **6** (명확한 MVVM + 단일 입력/출력) |

## 확장성

- 다운로드 엔진·플러그인(동영상/이미지) **모듈화**하기 좋은 단순 뷰
- **큐 + 진행률** UI를 덧씌우기 쉬움

## 장점

- **범용 URL 단일창** — 제품 콘셉 전달이 쉬움
- 의존성이 적어 **기능 붙이기**만 하면 됨

## 단점

- **실질 다운로드/분석 없음** — 데모·내부 PoC로만 쓰기 쉬움
- **법적(저작권) 안내** 없음 — 제품화 시 필수

## 제품으로서의 매력

**“URL 한 줄로 끝내는 다운로드 허브”**의 **MVP 셸**로 스토리가 팔립니다. **개발투자 대비** 콘셉만 보여 줄 때는 충분하나, **스토어/외배포**엔 **실기능+법무**가 선행돼야 합니다.
