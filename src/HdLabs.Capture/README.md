# HdLabs.Capture (EzLabs Capture)

**화면 캡처**를 목표로 한 WPF 셸입니다. ViewModel이 **Ready** 상태를 유지하는 **기능 미구현 스캐폴딩**입니다.

## 프로젝트 분석

- **실제 캡처:** Win32 DWM/GDI+·`System.Drawing`·Windows.Graphics.Capture·외부 캡처 라이브러리 등 **선택지**가 열려 있으나, 코드엔 **연결 없음**

## 기능 점수 (10점 만점)

| 항목 | 점수 |
|------|------|
| **현재** | **2~3** |
| **(기능 구현 가정)** | **7~8** (캡처 품질·핫키에 달림) |

## 확장성

- **전체 화면 / 영역 / 윈도우** 단계로 확장
- **클립보드 복사**·**자동 저장 경로**·**후처리(마킹)** 로 차별화

## 장점

- 시리즈 **비주얼 일관** — 번들(메모+캡처) **스위트** 내러티브 가능
- WPF + 공용 MVVM — **핫키 훅**만 붙이면 프로토타입 빠름

## 단점

- **기능 없음** — 스크린시 온보딩 의존
- **게임/DRM 화면** 캡처는 별도 기술·정책

## 제품으로서의 매력

**“가볍게 캡처 + 메모에 붙이기(연동)**” **워크플로**를 만들기 좋은 **피스**. 단독 제품이면 **ShareX/스니핑툴**과 **강한 경쟁** — **HD Labs 세트** 안에서만 의미가 살기 쉬움.
